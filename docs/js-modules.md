# Web UI JavaScript Modules

`www/app.js.min` is generated by `tools/build_web_assets.sh build`. Source modules live in `www/js/` and are concatenated in the order shown below. Keep shared state before modules that consume it, and keep `boot.js` last.

## www/i18n.json

Pure translation data embedded by `tools/build_web_assets.sh build` as `const I = <json>;` before runtime modules. Keep this file valid JSON: no JavaScript syntax, comments, or trailing commas.

## i18n-runtime.js

Language/theme helpers, startup gate, and authentication bootstrap.

- `applyLang()` applies `data-i` translations and language toggle text.
- `applyTheme()` applies the saved light/dark theme.
- `hex()` and `numberFromHex()` convert color values between numeric and CSS formats.
- `startup()`, `hideStartup()`, and `showStartup()` update the startup/auth overlay.
- `showDebugBuildBadge()` renders the debug build marker.
- `startApp()` applies UI state, activates the route, hides startup, and schedules preview startup.
- `initAuth()` checks auth state and either starts the app or shows the login form.

## state-i18n.js

Global DOM cache and shared app state used across all modules.

- `dbg()` is the no-op debug hook.
- Defines `E`, the central DOM element cache.
- Defines shared state such as `storeLoaded`, `libraryLoaded`, `filesLoaded`, `apps`, `settings`, `currentApp`, `authHeader`, and the authenticated `fetch` wrapper.

## cast-bootstrap.js

Top-level SPA navigation and generic sheet helpers.

- `wireKindTabs()` wires App/Live segmented tab buttons.
- `renderAppKindTabs()` renders and wires store/library kind tabs.
- `activate()` switches app panels and browser history paths.
- `footerExportAction()` creates/reuses the footer export button.
- `hideFooterExport()` hides the footer export button when dialogs change.

## cast-labels.js

## cast-files-install.js

- `loadExternalCastModule()` loads a Live app entry script.
- `openCastApp()`, `installCastApp()`, and `uninstallCastApp()` manage Live app lifecycle actions.

## runtime-transport.js

Singleton Live Runtime transport loaded after shared auth/state and before all runtime consumers. It bootstraps WebSocket v1 through authenticated `POST /api/runtime/session`, keeps the token in memory, sends binary protocol packets, owns preview/button subscriptions, and falls back to the lease-based HTTP runtime without changing the Live app API.

- Client packet types: `AUTH 0x01`, `CLAIM 0x02`, `RELEASE 0x03`, `DRAW_FRAME 0x04`, `SUBSCRIBE 0x05`, and `PING 0x06`.
- Server packet types: `READY 0x81`, `ACK 0x82`, `ERROR 0x83`, `SCREEN_RGB 0x84`, `BUTTON_EDGE 0x85`, `BUTTON_SNAPSHOT 0x86`, and `PONG 0x87`.
- Protocol version is `1`; request IDs, generations, and event sequences are big-endian.
- The bootstrap token is decoded to 16 raw AUTH bytes and is never placed in a URL or browser storage.
- `runtimeTransport.claim()`, `.frame()`, `.release()`, and `.enableButtons()` select exactly one transport.
- WebSocket requests have bounded ACK deadlines; a timeout closes the socket and discards stale queued frames before fallback.
- HTTP fallback claims are single-flight and a frame is never posted until the current owner has a lease.
- Button snapshots establish a big-endian uint32 sequence baseline; duplicate or older queued edges are ignored with wrap-safe comparison.
- Unload sends a lease-bearing HTTP release beacon when possible, otherwise closes WebSocket ownership for firmware-side disconnect release.

Final boot module. Must remain last in `tools/build_web_assets.sh`.

- Calls `bindUiHandlers()` and then `initAuth()` to start the app.
